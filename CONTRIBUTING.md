# Contributing to OPC Template

Thanks for your interest in contributing! This document explains how to contribute
code, issues, docs, and demos.

## Code of Conduct

Please read `CODE_OF_CONDUCT.md` and be respectful.

## How to contribute

1. Fork the repository.
2. Create a branch: `git checkout -b feat/short-description`.
3. Run tests: `make test`.
4. Commit with clear message, push and open a PR against `main`.

## PR requirements

- All CI checks must pass.
- Include test or docs for non-trivial changes.
- Keep PRs small and focused.

## Issues

- Search existing issues before opening a new one.
- Use templates: Bug Report / Feature Request.

## Local environment

- Require: Go (1.20+), Node (16+), Docker (optional)
- `make init` will bootstrap a minimal local environment.
